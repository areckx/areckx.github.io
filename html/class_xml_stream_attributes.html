Detailed Description

The XmlStreamAttributes class represents a vector of XmlStreamAttribute.

QtCore

Since
4.3 Attributes are returned by a XmlStreamReader in {XmlStreamReader::attributes()} {attributes()} when the reader reports a {XmlStreamReader::StartElement}{start element}. The class can also be used with a XmlStreamWriter as an argument to {XmlStreamWriter::writeAttributes()}{writeAttributes()}.

The convenience function value() loops over the vector and returns an attribute value for a given namespaceUri and an attribute's name.

New attributes can be added with append().

◆ hasAttribute() [1/3]

bool XmlStreamAttributes::hasAttribute ( const QString &  qualifiedName) const
inline
Since
4.5

Returns true if this XmlStreamAttributes has an attribute whose qualified name is qualifiedName; otherwise returns false.

Note that this is not namespace aware. For instance, if this XmlStreamAttributes contains an attribute whose lexical name is "xlink:href" this doesn't tell that an attribute named href in the XLink namespace is present, since the xlink prefix can be bound to any namespace. Use the overload that takes a namespace URI and a local name as parameter, for namespace aware code.

◆ value() [4/5]

QStringRef XmlStreamAttributes::value ( const QString &  qualifiedName) const

This is an overloaded member function, provided for convenience. It differs from the above function only in what argument(s) it accepts.

Returns the value of the attribute with qualified name qualifiedName , or an empty string reference if the attribute is not defined. A qualified name is the raw name of an attribute in the XML data. It consists of the namespace prefix, followed by colon, followed by the attribute's local name. Since the namespace prefix is not unique (the same prefix can point to different namespaces and different prefixes can point to the same namespace), you shouldn't use qualified names, but a resolved namespaceUri and the attribute's local name.
