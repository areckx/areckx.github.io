1 # MuseScore: Music notation and composition software
2 
3 MuseScore is an open source and free music notation software.
4 For support, contribution, bug reports, visit [MuseScore.org](https://musescore.org). Fork and make pull requests!
5 
6 ## Features
7 
8 * WYSIWYG design, notes are entered on a "virtual notepaper"
9 * TrueType font(s) for printing & display allows for high quality scaling to all sizes
10 * easy & fast note entry
11 * many editing functions
12 * MusicXML import/export
13 * Midi (SMF) import/export
14 * MuseData import
15 * Midi input for note entry
16 * integrated sequencer and software synthesizer to play the score
17 * print or create pdf files

28 ## Packages
29 * **aeolus** Clone of [Aeolus](http://kokkinizita.linuxaudio.org/linuxaudio/aeolus/)
30 Disabled by default in the stable releases. See http://dev-list.musescore.org/Aeolus-Organ-Synth-td7578364.html
31 Kept as an example of how to integrate with a complex synthesizer.
32 
33 * **assets** Graphical assets, use them if you need a MuseScore icon. For logo, color etc... see https://musescore.org/en/about/logos-and-graphics
34 
35 * **awl** Audio Widget Library, from the MusE project
36 
37 * **build** Utility files for build
38 
39 * **bww2mxml** Command line tool to convert BWW files to MusicXML. BWW parser is used by MuseScore to import BWW files.
40 
41 * **demos** A few MuseScore files to demonstrate what can be done
42 
43 * **fluid** Clone of [FluidSynth](https://sourceforge.net/projects/fluidsynth/), ported to C++ and customized
44 
45 * **fonts** Contains fontforge source (sfd) + ttf/otf fonts. MuseScore includes the "Emmentaler" font from the Lilypond project.
46 
47 * **libmscore** Data model of MuseScore
48 
49 * **mscore** Main code for the MuseScore UI
50 
51 * **mstyle** Clone of KDE4 style Oxygen
52 
53 * **msynth** Abstract interface to Fluid + Aeolus
54 
55 * **mtest** Unit testing using QTest
56 
57 * **omr** Optical music recognition
58 
59 * **share** Files moved to /usr/share/... on install
60 
61 * **test** Old tests. Should move to mtest
62 
63 * **vtest** Visual tests. Compare reference images with current implementation
64 
65 * **thirdparty** Contains projects which are included for convenience, usually to integrate them into the build system to make them available for all supported platforms.
66 
67  * **thirdparty/rtf2html**

72 
73  * **thirdparty/ofqf**

75 
76  * **thirdparty/singleapp**
77  Clone from [Qt Single Application](https://github.com/qtproject/qt-solutions/tree/master/qtsingleapplication)
78 
79  * **thirdparty/portmidi**
80  Clone from [PortMidi](https://sourceforge.net/projects/portmedia/)

82  * **thirdparty/beatroot**

87 ## Building
88 **Read the developer handbook for a [complete build walkthrough](https://musescore.org/en/developers-handbook/compilation) and a list of dependencies.**
89 
90 ### Getting sources
91 If using git to download repo of entire code history, type:
92 
93  git clone https://github.com/musescore/MuseScore.git
94  cd MuseScore
95 
96 Else can just download the latest source release tarball from https://github.com/musescore/MuseScore/releases, and then from your download directory type:
97 
98  tar xzf MuseScore-x.x.x.tar.gz
99  cd MuseScore-x.x.x
100 
101 ### Release Build
102 To compile MuseScore, type:
103 
104  make release
105 
106 If something goes wrong, then remove the whole build subdirectory with `make clean` and start new with `make release`.
107 
108 ### Running
109 To start MuseScore, type:
110 
111  ./build.release/mscore/mscore
112 
113 The Start Center window will appear on every invocation, until you disable that setting via the "Preferences" dialog.
114 
115 ### Installing
116 To install to default prefix using root user, type:
117 
118  sudo make install
119 
120 ### Debug Build
121 A debug version can be built by doing `make debug` instead of `make release`.
122 
123 To run the debug version, type:
124 
125  ./build.debug/mscore/mscore
126
